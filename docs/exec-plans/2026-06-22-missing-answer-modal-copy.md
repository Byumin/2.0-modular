# 미응답 확인 모달 문구 수정

## 요청 요약
- 수검자 문항 화면에서 미응답 문항이 있을 때 뜨는 확인 모달의 제목, 설명, 버튼 문구를 변경한다.
- 설명 문구 중 `n개 문항` 부분은 굵게 표시한다.

## 작업 목표
- 미응답 확인 모달의 문구를 요청안과 일치시킨다.
- 제출/이동 로직은 변경하지 않는다.
- 프론트엔드 빌드로 문법 오류 여부를 확인한다.

## 초기 가설
- 미응답 확인 모달은 수검자 문항 단계 컴포넌트에서 렌더링된다.
- 정적 문구 변경이므로 백엔드 API나 DB 변경은 필요 없다.

## 실행 계획
1. `AGENTS.md`, `DESIGN.md`, 실행계획 규칙을 확인한다.
2. 미응답 확인 모달 렌더링 위치를 확인한다.
3. 모달 제목, 안내문, 버튼 라벨을 변경한다.
4. 프론트엔드 빌드를 실행해 검증한다.
5. 결과와 미검증 항목을 기록한다.

## 작업 중 변경 사항
- `frontend/src/pages/assessment/steps/QuestionStep.tsx`의 미응답 확인 모달 문구를 변경했다.
- 안내 문구는 두 줄로 분리하고, `n개 문항` 부분만 `strong`으로 굵게 표시했다.
- 버튼 라벨을 `첫 미응답 이동`, `제출하기`로 변경했다.

## 결과
- 요청한 문구와 버튼 라벨이 반영됐다.
- 제출 로직과 미응답 이동 로직은 변경하지 않았다.

## 검증 내용
- `npm run build:frontend` 통과.
- `frontend/dist/assets/index-CdT6OVn2-v2.js`에서 새 문구 포함 확인.
- 브라우저 스크린샷 검증은 이 환경에서 Playwright 브라우저 실행 조건이 없어 수행하지 못했다.

## 회고
- 단순 카피 변경으로 계획 문제나 실행 판단 문제는 확인되지 않았다.
