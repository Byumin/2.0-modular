Interactive Flow · Admin Auth · v2

관리자 로그인 · 인증 · 세션 생성

페이지 진입부터 SHA-256 비밀번호 검증, 메모리 세션 저장, HttpOnly 쿠키 발급, 후속 인증까지 실제 코드 기반 런타임 흐름을 단계별로 탐색합니다. 그래프 노드를 클릭하면 내부 서브노드와 상세 데이터가 펼쳐집니다.
※ 예시 데이터는 설명용 샘플입니다.

시나리오 선택

정상 로그인 경로 — 페이지 진입 → 폼 제출 → 스키마 검증 → SHA-256 비교 → 토큰 생성 → HttpOnly 쿠키 설정까지 전체 9단계 흐름이 활성화됩니다.

핵심 구조

  • 세션은 DB가 아닌 서버 메모리(ADMIN_SESSIONS dict)에 저장됩니다.
  • DB 접근은 SQLAlchemy ORM이 아닌 sqlite3 raw connection(modular.db)을 사용합니다.
  • 비밀번호는 hashlib.sha256으로만 해시되며 salt가 없습니다.
  • 쿠키는 HttpOnly + SameSite=Lax + Max-Age 8h로 발급됩니다.
