커스텀 검사 생성 흐름

관리자 생성 모달에서 커스텀 검사를 만들 때, 브라우저는 먼저 추가 인적사항과 검사별 척도 선택을 따로 정리하고 누락 실시구간이 있으면 확인 모달로 한 번 더 분기합니다. 서버에 들어간 뒤에도 라우터, Pydantic, 관리자 인증, 추가 인적사항 재정규화, parent variant 조회, 척도 교집합 계산, 저장 구조 조립이 각각 독립 단계로 실행되고, 마지막 목록 화면은 `selected_scales_json`을 다시 해석해 카드 shape를 복원합니다.

시나리오 선택

핵심 구조

  • 페이지 로드 직후 카탈로그와 관리 목록이 먼저 조회되고, 생성 모달은 그 카탈로그를 로컬 상태로 사용합니다.
  • 프런트 단계도 하나로 묶지 않고, 추가 인적사항 정규화와 검사별 `test_configs` 조립을 분리해서 보여줍니다.
  • 서버 단계도 인증, 필드 재정규화, parent row 조회, variant 대조, 저장 구조 조립을 각각 분리했습니다.
  • 저장은 `selected_scales_json` 중심 구조이고, 이후 목록 조회는 이 값을 다시 파싱해 `test_configs`로 복원합니다.
