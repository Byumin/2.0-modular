관리자 로그인 검증과 세션 생성

관리자 로그인 화면 진입부터 기존 세션 확인, 로그인 요청, 스키마 검증, 관리자 조회, SHA-256 비밀번호 비교, 메모리 세션 저장, HttpOnly 쿠키 발급, 후속 인증 재사용까지의 실제 런타임 흐름을 단계별로 설명한다.

시나리오 선택

핵심 구조

  • /api/admin/me로 기존 세션을 먼저 확인한다.
  • AdminLoginIn이 요청 body 최소 길이를 검증한다.
  • admin_user 조회는 루트 modular.db 직접 접근이다.
  • 로그인 성공 세션은 DB가 아니라 ADMIN_SESSIONS 메모리 딕셔너리에 저장된다.
  • 최종 인증 토큰은 admin_session HttpOnly 쿠키로 브라우저에 전달된다.

런타임 그래프

노드를 클릭하면 내부 함수와 요소가 큰 노드 안에서 펼쳐집니다. 확대/축소로 전체 흐름과 세부 흐름을 오가며 볼 수 있습니다.
